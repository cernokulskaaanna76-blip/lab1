# Lab 3 — SQLite Backend

## Запуск проекту

1. Встановити залежності:
npm install

2. Запустити сервер:
npm run dev

Сервер буде доступний:
http://localhost:3000

---

## База даних

- Використовується SQLite
- Файл БД: data/app.db
- База створюється автоматично при запуску

---

## Ініціалізація

При запуску:
- відкривається база даних
- створюються таблиці
- застосовуються міграції

---

## Seed (тестові дані)

npm run seed

---

## API приклади

### Створення користувача
POST /api/users

### Отримання списку
GET /api/shifts

### Фільтрація
GET /api/shifts?status=planned

### Статистика
GET /api/shifts/stats/summary

---

## Схема БД

Таблиці:
- Users
- Shifts
- Swaps

Зв'язки:
- User → Shifts (1:N)

Обмеження:
- NOT NULL
- UNIQUE
- FOREIGN KEY

---

## Особливості

- Реалізовано CRUD
- Є фільтрація (WHERE)
- Є агрегація (COUNT)
- Є міграції (schema_migrations)
- Є обробка помилок (400, 404, 409)
- SQL винесений в repository

---

## SQL Injection (демо)

При передачі:
' OR 1=1 --

можна отримати всі записи.

Це демонструє небезпеку конкатенації SQL.
